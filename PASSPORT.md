## 2. Назначение

ZFS — высокопроизводительная файловая система и менеджер томов с функциями защиты данных, предназначенная для использования в составе серверной и инфраструктурной программно-аппаратной платформы.

Область применения:
- серверные операционные системы общего назначения;
- системы хранения данных (СХД);
- виртуализационные платформы;
- резервное копирование и архивирование данных;
- объекты критической информационной инфраструктуры (КИИ).

## 3. Функциональные характеристики

| Функция | Описание |
|---------|----------|
| Проверка целостности | Контрольные суммы каждого блока (SHA-256, BLAKE3, Fletcher-4) |
| RAID | Программный RAID-Z (аналог RAID-5/6/7) |
| Снапшоты | Мгновенные снимки файловой системы без копирования данных |
| Шифрование | AES-256-GCM, AES-256-CCM на уровне датасета |
| Сжатие | LZ4, ZSTD, gzip, lzjb |
| Дедупликация | Поблочная дедупликация данных |
| Квоты | Гранулярное ограничение дискового пространства |
| Клоны | Клонирование датасетов и снапшотов |
| Репликация | Инкрементная репликация (zfs send/receive) |

## 9. Тестирование

Репозиторий содержит комплексный набор тестов (>1100 тест-кейсов), включающий:
- функциональные тесты файловой системы;
- тесты устойчивости (zloop);
- тесты производительности;
- тесты шифрования и контроля целостности.

Тестирование проводится в автоматическом режиме с использованием CI/CD.
